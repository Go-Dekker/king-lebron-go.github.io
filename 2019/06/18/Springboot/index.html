Springboot

2019-06-18
Springboot 使用logback作为默认的日志框架 ，支持 Java Util Logging, Log4j2, Lockback 等作为日志框架。
SpringBoot 实现热部署主要有两种方式 : Spring Loaded , Spring-boot-devtools。
SpringBoot 读取配置文件的方式 : @Value , @ConfigurationProperties , @PropertySource+@Value , @PropertySource+@ConfigurationProperties , Environment 。

Springboot 优缺点

优点
  • 易上手，xml 配置 使用 configuration 替代 ,pom.xml中各种starter直接引入就可集成，基本上可以做到自动化配置。
  • 简化项目启动部署，Spring Boot微服务，仅仅一个jar包就能把整个应用部署启动 。
  • 自带 tomcat、jetty 容器 。
  • 引入spring-boot-start-actuator的依赖，就可以实现对服务性能的监控 。
  • 支持关系数据库和非关系数据库 。
缺点
  • Spring boot 默认使用 spring data的 Jpa进行sql查询，但是遇到复杂的sql，还是写原生sql比较好，看上去更加直观 。
  • jsp没法用，当你引入了依赖就会自动帮你配置，会造成一些错误配置，造成启动失败 。
  • 集成度很高，不易于排错。

Springboot 常用注解

@SpringBootApplication
包含了@ComponentScan、@Configuration和@EnableAutoConfiguration注解。其中@ComponentScan让spring Boot扫描到Configuration类并把它加入到程序上下文。
@Configuration
等同于spring的XML配置文件注入Bean 的操作。
@EnableAutoConfiguration
自动装配。
@ComponentScan
组件扫描，可自动发现和装配一些Bean。
@Component
装配Bean。
@RestController
@Controller和@ResponseBody的合集,表示这是个控制器bean,并且是将函数的返回值直 接填入HTTP响应体中,是REST风格的控制器。
@Autowired
自动导入。
@PathVariable
获取参数。
@Import
用来导入其他配置类。
@ImportResource
用来加载xml配置文件 ，使用 @ImportResource 注解导入老 Spring 项目配置文件来兼容老的项目 ， 例如 @ImportResource({“classpath:dubbo.xml”})。
@Service
一般用于修饰service层的组件。
@Repository
使用@Repository注解可以确保DAO或者repositories提供异常转译，这个注解修饰的DAO或者repositories类会被ComponetScan发现并配置，同时也不需要为它们提供XML配置项。
@Bean
用@Bean标注方法等价于XML中配置的bean。
@Value
注入Spring boot application.properties配置的属性的值。
@MapperScan
mybatis 配置 扫描所有dao包的路径或者在没个dao类上面加上@Mapper注解。
@RequestMapping
处理请求地址的映射，类上的此注解是方法上的父地址
@Profile
此注解加载类上 表示此类对 application.yml中哪个环境起作用 例如 @Profile({“dev”,”test”,”uat”})
@Aspect
此注解加在类上，表示此类为切面类。
@Pointcut
定义切点 例如 使用自定义注解作为切点 @Pointcut(“@annotation(com.gbicc.credit.houseAppraisal.annotation.Log)”)
@Before @After @Around
切面通知类型 分别对应 前置通知、后置通知、环绕通知。
